CONCEPTOS DE GIT

1. ¿Qué es un control de versiones?

Un control de versiones es un sistema que permite realizar un seguimiento de los cambios realizados en un conjunto de archivos a lo largo del tiempo. Esto permite revertir cambios, colaborar con otros y mantener un historial detallado de modificaciones.

2. ¿Qué es GIT?

GIT es un sistema de control de versiones distribuido, diseñado para manejar todo tipo de proyectos con rapidez y eficiencia. Es especialmente útil para el trabajo en equipo y la gestión de proyectos de software.

3. ¿Qué es un repositorio?

Un repositorio es un espacio donde se almacena un proyecto y todas sus versiones. Puede ser local (almacenado en tu propia máquina) o remoto (almacenado en un servidor, accesible a través de internet).

4. ¿Qué es GITHUB?

GITHUB es una plataforma de alojamiento de repositorios GIT. Permite gestionar proyectos, colaborar con otros desarrolladores y alojar código fuente, además de proporcionar herramientas adicionales para la integración y el despliegue continuo.

5. ¿Qué es una rama en GIT y para qué sirve?

Una rama en GIT es una línea paralela de desarrollo que permite trabajar en características o cambios independientes sin afectar la rama principal (generalmente llamada "main" o "master").

6. ¿Cuáles son los estados en GIT?

En GIT, un archivo puede estar en uno de tres estados: modificado (modificado pero no preparado para commit), preparado (marcado para ser incluido en el próximo commit) y confirmado (almacenado en el historial del repositorio).

7. ¿Qué es la terminal, consola o línea de comandos y para qué sirve?

La terminal, consola o línea de comandos es una interfaz de texto que permite a los usuarios interactuar con el sistema operativo o el software mediante comandos. Es útil para realizar tareas de administración, automatización y desarrollo.

8. Escribir el funcionamiento de cada uno de los siguientes comandos:

  • ls: Lista los archivos y directorios en el directorio actual.
  • cd <directorio>: Cambia al directorio especificado.
  • cd ..: Sube un nivel en la jerarquía de directorios.
  • mkdir <nombre>: Crea un nuevo directorio con el nombre especificado.
  • touch <nombre>: Crea un archivo vacío con el nombre especificado.
  • rm <nombre>: Elimina el archivo especificado.
  • cp <nombre> <directorio>: Copia el archivo especificado al directorio indicado.
  • mv <nombre> <directorio>: Mueve el archivo especificado al directorio indicado.

9. Realizar un listado con el nombre y el funcionamiento de los comandos vistos en la sesión de clase de GIT.

  • git --version: Mostrar la version de GIT instalada en el equipo.
  • git config --global user.name "[name]": Establece el nombre que deseas esté anexado a tus transacciones de commit.
  • git config --global user.email "[email address]": Establece el e-mail que deseas esté anexado a tus transacciones de commit.
  • git config user.name: Mostar el usuario
  • git config user.email: Mostrar el correo electronico
  • git init [project-name]: Crea un nuevo repositorio local con el nombre especificado.
  • git clone [url]: Descarga un proyecto y toda su historia de versión.
  • git add [file]: Toma una instantánea del archivo para preparar la versión.
  • git status: Enumera todos los archivos nuevos o modificados que se deben confirmar.
  • git commit -m "[descriptive message]": Registra las instantáneas del archivo permanentemente en el historial de versión.
  • git log: Enumera el historial de la versión para la rama actual.
  • git push [alias] [branch]: Carga todos los commits de la rama local al GitHub.
